# 首页装修与发布模型说明

## 1. 操作模型

首页装修采用三层状态，避免“边改边影响生产”：

1. **未保存编辑**：只存在当前浏览器；离开页面会提示。
2. **已保存草稿**：写入 `home_sections` 工作区，后台预览可见，线上访客不可见。
3. **线上版本**：仅来自 `home_page_releases` 不可变快照，由发布操作一次性切换。

后台顶部状态条会明确显示：

- 有未保存编辑
- 草稿待发布
- 草稿与线上一致
- 当前线上版本号、版本说明与发布时间

标准操作顺序是：**编辑 → 保存草稿 → 私密预览 → 发布到线上**。

## 2. Hero 与 Banner

Hero 是首页最上方的“首屏主视觉区块”，Banner 是它使用的图片、标题和按钮素材。两者职责不同，但运营操作已合并到首页装修页：

- Hero 区块只保存一个 `banner_id`，不再显示无意义的 HERO/SECONDARY 位置选择。
- 可在 Hero 属性面板直接选择、创建和编辑主视觉素材。
- 首页发布时会把选中 Banner 的图片和 EN/ES/FR 文案冻结进版本快照。
- 之后修改或下线该 Banner，不会改变已经发布的首页版本。
- 发布首页也不会反向修改 Banner 自身状态。

因此 Banner 的“草稿/已发布/已下线”主要服务于素材管理；首页是否上线以整页发布版本为准。

## 3. 私密预览

点击“私密预览”会先保存草稿，再生成一个 30 分钟有效的预览快照：

- 原始令牌为 32 字节随机数，数据库只保存 SHA-256。
- 预览内容在签发时冻结；之后继续编辑，不会偷偷改变已打开的预览。
- 商城验证令牌后写入 HttpOnly、SameSite Cookie，并立即移除 URL 中的令牌。
- 预览响应使用 `private, no-store`、`noindex` 和 `no-referrer`。
- 预览页有明显标识和“退出预览”入口。

私密预览适合日常内容验收，不需要部署一套完整测试环境。测试环境仍适合验证代码升级、接口变更、支付等系统级改动；两者不是替代关系。

## 4. 原子发布

发布会在同一事务中完成：

1. 锁定首页工作区。
2. 校验至少一个启用区块、Hero 数量和 Banner 引用。
3. 校验草稿指纹，防止确认期间被其他管理员修改。
4. 生成配置快照和 EN/ES/FR 三份最终渲染快照。
5. 插入单调递增的新版本。
6. 原子切换 `site_builder_config.active_home_release_id`。

生产首页只读当前活动版本，不回退读取草稿；即使活动版本异常缺失，也会返回空动态区块而不是泄漏未发布内容。

## 5. 版本与回滚

- 每次发布生成新的 `V1`、`V2`……不可变版本。
- 版本记录包含说明、发布时间、发布管理员和回滚来源。
- 回滚不会覆盖历史记录，而是复制目标快照生成一个新的发布版本。
- 回滚同时将目标配置恢复为草稿工作区，保证后台所见与线上一致。

## 6. 区块配置

### ProductRail

- 新品：推荐服务的 `new_arrivals`
- 畅销：推荐服务的 `best_sellers`
- 人工推荐：可搜索选择商品，发布时过滤已下架商品并保持选择顺序
- 按分类：选择分类并设置最新、价格或推荐顺序

### ThemeCards

- 自动：按根分类顺序展示并限制数量
- 手动：可选择任意分类，并按选择顺序展示

### EditorialFeature

按婚礼日期从新到旧展示。当前数据没有可靠人气指标，因此不提供会误导运营结果的“人气/随机”选项。

### Newsletter

支持 EN/ES/FR 的眉标题、标题、说明、占位符和按钮文案。

### Custom

支持多语言标题、副标题、正文，以及图片上传、按钮文案和链接；商城已实现相应渲染。

## 7. 初始数据

初始化程序按业务键幂等补齐：

- 原型首页的 Hero、主题分类、商品、新人案例和订阅区块。
- 一个线上 Hero Banner 与一个草稿 Hero 示例。
- 旧 Hero 若没有明确选择，会绑定当前首个有效 Hero Banner。
- 已存在的运营数据不会被覆盖。

## 8. 关键接口

- `PUT /api/admin/site-builder/home-sections/draft`：整页保存草稿
- `GET /api/admin/site-builder/home-publication/preview`：后台草稿画布
- `POST /api/admin/site-builder/home-publication/preview-token`：创建私密预览
- `POST /api/admin/site-builder/home-publication/publish`：带草稿指纹发布
- `GET /api/admin/site-builder/home-publication/history`：版本记录
- `POST /api/admin/site-builder/home-publication/releases/{id}/rollback`：回滚并重新发布
- `GET /api/store/content/home`：生产首页，只读活动版本
- `GET /api/store/content/home/preview`：令牌预览快照

## 9. 验收重点

- 保存草稿后，匿名生产首页响应不变化。
- 私密预览能看到草稿且链接过期后不可访问。
- 发布后 EN/ES/FR 同时切换到同一版本。
- 发布确认期间发生并发编辑时，发布被拒绝并要求重新预览。
- 修改 Banner 后，历史首页版本保持不变。
- 回滚生成新版本，且草稿工作区同步恢复。
