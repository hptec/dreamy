## L1.3 设计约束（传递给下一阶段）

### 数据流设计关注点
1. **AI 翻译代理流程**：前端翻译按钮 → 后端代理 → 外部网关 → 组装 prompt（系统前缀+术语表+用户要求）→ 记录日志 → 返回译文
2. **模型同步流程**：配置保存/手动刷新/定时任务 → 调用 /v1/models → 更新 model_list + models_synced_at
3. **邮件模板选择流程**：触发邮件事件 → 按 user.locale_pref → order.locale_snapshot → EN 优先级选模板 → 渲染发送

### 错误策略设计关注点
1. **API Key 加密/解密失败**：422202（加密异常）需回滚事务，不落库
2. **外部网关调用失败**：502301（AI 网关）/ 502201（测试连接）需重试 1 次，记录详细错误原因
3. **翻译失败容错**：前端 toast 提示但不阻塞保存流程（决策 10）
4. **模型同步失败降级**：连续失败 3 次后降级为 manual 并发告警（决策 5）

### 共享约束
- **术语表注入格式**：启用术语按 category 分组，每组不超过 50 条（避免 prompt 过长）
- **翻译请求超时**：30s（ai-translation-api）
- **测试连接超时**：10s（gateway-api）
- **调用记录保留期**：90 天（决策 7），定时任务每日凌晨 3:00 批量删除（分批 LIMIT 5000）
