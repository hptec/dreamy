# Feature Map — portal-admin 运营管理后台

> Dreamy 平台后台完整功能表。门户 ID：`portal-admin` | 端口：5174
> 技术栈：**pnpm + Vue3 + Vite + Tailwind + Headless UI**（与前台 Next.js 有意割裂，设计 token 同源）
> 与 portal-store 完全隔离，独立鉴权（RBAC）。来源：requirements-brief.md + mall-admin 参考
>
> **定位**：传统电商后台（mall-admin 参考）+ 全站内容可配置（CMS）+ 静态站点发布器（SSG）三合一。
> 已于 2026-05-30 全量实现 22 个页面，运行验证 0 错误。

## 设计创新：相比标准 mall-admin 的差异化

| 模块 | mall-admin 原范式 | Dreamy 增强 |
|------|------------------|------------|
| 站点装修 | 仅「广告列表」 | **首页装修**（可视化区块编排 + 实时预览）+ **导航/页脚/公告配置** |
| 发布机制 | 后台直连数据库实时生效 | **发布中心**：改动 diff → 模拟 next build → 列出受影响 HTML 页面 → 发布历史 + 回滚 |
| 内容范围 | 商品/订单为主 | **全站内容可配**：Hero/推荐位/Mega Menu/Footer/Banner/Blog/RealWedding/Lookbook/Guide |

- **粒度**：菜单级（一个路由 = 一个权限点，共 20 个菜单权限点）
- **模式**：二元（有权限 → 菜单可见 + 路由可访问；无权限 → 菜单隐藏 + URL 拦截）
- **角色**：5 个预设角色 + N 个自定义角色；超管不可变
- **数据范围**：本轮无数据级隔离（Future 预留 scope 字段）

## 迭代 3：AM12 身份与认证（消费者账户管理 · 2026-05-31）

> 配合消费端 passwordless（邮箱验证码 + Google + Apple）落地。后台员工登录**保持邮箱+密码不变**；本模块是后台对**消费者身份**的管理能力。已有最大编号 A-063（Won't），本轮新增自 A-064 起。

### 功能表

| 编号 | 功能名 | 优先级 | 关键交互 | 承载页面 |
|------|--------|--------|---------|---------|
| A-064 | 客户登录方式卡 | Must | 客户详情展示已绑定方式（邮箱已验证 / Google / Apple-relay）、绑定时间、各方式最近登录；Apple 标注 relay 邮箱 + sub 主键 | CustomerDetail |
| A-065 | 客户登录记录 + 强制下线 | Must | 登录记录表（时间/方式/IP/设备/位置/成功失败）+ 活跃会话列表 + 单会话下线 + 强制下线全部（passwordless 无需重置密码） | CustomerDetail |
| A-066 | 账户合并工具（identity merge） | Must | 重复账户候选（依据：已验证邮箱 / Apple sub / 手机号）+ 合并预览（保留方可对调，订单/消费/登录方式归集）+ 确认 + 空态 | CustomerMerge（`/customers/merge`） |
| A-067 | 登录与认证配置 | Must | 开关 Google/Apple（邮箱主登录锁定）、OTP 长度/有效期/重发/尝试上限、按已验证邮箱自动归并 + 冲突提示 + 最少登录方式数、Google/Apple OAuth 凭据占位 | AuthSettings（`/system/auth`） |
| A-068 | 操作日志扩展 | Must | 新增动作类型：Google/Apple 登录、账户合并、强制下线、认证配置变更；合并/配置变更带变更前后对比 | OperationLogs（mock 扩展，组件复用） |

### 权限与菜单
- 新增 2 个菜单权限点：`/customers/merge`（用户管理组）、`/system/auth`（系统管理组），纳入 menuPermissionKeys（共 22 点），超管自动全量。
- 侧边栏：用户管理组加「账户合并」；系统管理组加「登录与认证」（角色权限与操作日志之间）。
- 路由：新增 `/customers/merge`、`/system/auth`，受现有 A-049 路由守卫 + A-050 侧边栏动态渲染统一管控。

### 安全设计要点（与消费端一致）
- 一个人多凭证：`(provider, provider_uid)` → user_id；不以 email 为主键。
- 仅 `email_verified=true` 自动归并；冲突提示而非静默合并（防劫持）。
- Apple Hide My Email：relay 邮箱不可作匹配键，以 `sub` 为稳定主键，首次授权即落库。
- passwordless ⇒ 后台对消费者「重置密码」无意义，替换为「强制下线全部会话」。
