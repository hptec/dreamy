# 功能地图 — portal-admin 迭代 2：RBAC 权限体系

> Dreamy 运营管理后台 RBAC 权限体系功能扩展。门户 ID：`portal-admin` | 端口：5174
> 技术栈：pnpm + Vue 3 + Vite + Tailwind CSS + Headless UI
> 来源：requirements-brief.md 迭代 2 章节 + feature-map-portal-admin.md（AM1-AM10）
> 扩展日期：2026-05-30

## 概述

- **核心想法**：将 A-040（系统设置）的 3 个子 Tab 拆分为 3 个独立菜单页（管理员管理 / 角色权限 / 操作日志），权限模型从"full/read/none 三级只读展示"升级为"二元复选框矩阵 + 路由守卫 + 侧边栏动态渲染"的实效 RBAC 体系。
- **核心实体**：管理员账号（Admin Account）、角色（Role）、权限点（Permission/Menu Item）、操作日志（Operation Log）
- **影响范围**：新增 AM11「系统管理」模块，替换原 AM9「系统」（A-040 废弃）；侧边栏菜单重组；全局路由守卫与侧边栏渲染逻辑升级
- **已有功能最大编号**：A-040（另有 CFG-HOME、CFG-NAV、SSG-PUBLISH 三个特殊编号）
- **本轮新增编号**：A-041 起

## 扩展记录

### 各维度扩展明细

#### 维度一：CRUD 完整性

**管理员账号**
- Create：弹窗新增，字段（姓名/邮箱/密码/角色/状态），邮箱唯一校验，密码显示切换
- Read：列表页含搜索（姓名/邮箱模糊）、筛选（角色/状态下拉）、分页、排序；空数据态引导
- Update：弹窗编辑（邮箱只读不可改），角色降级自保护校验；Toggle 状态切换 + 重置密码
- Delete：确认弹窗 + 自身保护 + 超管保护
- Batch：批量启用/禁用/删除（Should Have A-055）

**角色**
- Create：新增自定义角色（名称 + 权限矩阵）
- Read：角色列表（类型标签/管理员计数/权限计数）
- Update：权限矩阵编辑器（复选框网格），预设角色可编辑权限不可改名称，超管全部锁定
- Delete：自定义角色可删除（含管理员迁移提示），预设角色不可删除

**操作日志**
- Read Only：列表（多维筛选 + 分页）+ 详情抽屉（变更前后对比）
- 无 Create/Update/Delete——系统自动记录，管理员不可人为操作
- Export：CSV 导出（Should Have A-054）

#### 维度二：权限与角色

- **角色划分**：超管（全部权限 + 不可变）/ 4 个预设角色（权限可调）/ N 个自定义角色（完整 CRUD）
- **权限粒度**：菜单级二元模型（有/无）。一个菜单项（路由）= 一个权限点。不区分读写——有菜单权限即可查看和操作
- **权限效果**：① 侧边栏可见性（A-050）② 路由可访问性（A-049）
- **数据隔离**：本轮无数据级隔离（所有同角色管理员看到相同数据），Future 预留 scope 字段
- **审批流**：本轮无审批流（删除、禁用等操作即时生效）
- **操作日志**：敏感操作自动记录（A-052/A-053），不可删除

#### 维度三：数据流转

- **管理员数据**：输入来源 = 超管/有权限管理员手动新增；输出去向 = 无（本轮无导出）
- **角色权限数据**：输入 = 管理员在矩阵中勾选配置；输出 = 运行时被 A-049/A-050 消费
- **操作日志数据**：输入 = 系统自动记录（管理员执行操作时写入）；输出 = CSV 导出（A-054）
- **通知触发**：本轮无通知（密码重置仅为前端演示，无邮件发送）。Future 预留邮件服务
- **数据联动**：
  - 删除角色 → 检查该角色下管理员数 → 提示迁移或确认
  - 修改权限 → A-049 路由守卫 + A-050 侧边栏即时生效（刷新页面后或下次路由跳转时）
  - 禁用/删除管理员 → 其当前登录态处理（原型假设：下次请求时拦截）

#### 维度四：异常与边界

- **空数据态**：
  - 管理员列表为空 → "暂无管理员"引导（A-041 已涵盖）
  - 无自定义角色 → "尚未创建自定义角色"提示（A-046 已涵盖）
  - 操作日志为空 → "暂无操作日志"（A-052 已涵盖）
  - 日志筛选无结果 → "未找到匹配的操作日志，请调整筛选条件"
- **极值场景**：
  - 大量操作日志（10万+）→ 分页 + 筛选缩小范围 + 导出上限 10000 条
  - 大量菜单项（50+）→ 权限矩阵按分组折叠/展开
- **并发冲突**：
  - 多人同时编辑同一角色权限 → 保存时提示"权限已被他人修改，请刷新后重试"（乐观锁预留）
  - 编辑中的管理员被他人删除 → 提交时后端返回 404，前端提示"该管理员已不存在"
- **网络异常**：
  - 权限矩阵保存失败 → toast 错误提示 + 保留表单内容不丢失
  - 路由守卫中权限数据加载失败 → 降级为拒绝访问（安全优先）
- **数据不一致**：
  - 已删除角色的管理员登录 → 分配默认"数据分析"角色（最小权限） + 提示联系超管
  - 菜单项被移除但角色权限仍引用 → 权限矩阵自动清理无效项

#### 维度五：配套能力

- **搜索与筛选**：
  - 管理员列表：姓名/邮箱模糊搜索 + 角色/状态下拉筛选
  - 操作日志列表：操作人搜索 + 操作类型多选 + 时间范围 DateRangePicker
- **帮助引导**：
  - 首次进入角色权限页 → 顶部信息条"提示：勾选的菜单项将对应该角色可见和可访问"
  - 操作日志详情 → 变更对比表格使用颜色图例（绿=新增/红=删除/黄=修改）
- **配置管理**：
  - 权限预设模板（A-056 Should Have）→ 快速加载标准权限集
- **审计追踪**：
  - 所有管理员操作自动记录日志（A-052）→ 谁在什么时间从哪个 IP 做了什么操作
