# 后端流程推断

> 本文件为 Phase 2.3 LLM 推断产物，作为 Phase 2.3.1 维度激活的信号源，不作为下游直接消费。
> 推断结论需经 Phase 2.3.1 用户确认后，沉淀到 decision.md 的「后端关键决策」章节。

## 识别到的业务模式

- 商品/分类/内容/Banner：典型 CRUD + 状态发布（draft/published/archived），读多写少
- 订单：多状态工作流（待支付 → 已支付 → 发货 → 完成/退款），含外部支付集成
- 购物车：登录态持久化 + 匿名合并
- 评价：提交 → 审核（pending/approved/rejected）工作流
- 看板：跨实体聚合查询（GMV/漏斗/流量）

## 推断的后端机制

- **[事务]** 下单需原子完成：订单创建 + 现货 SKU 库存扣减（乐观锁防超卖）+ 购物车清空；退款审核通过需原子完成退款记录 + 订单状态变更 + 现货库存回补
- **[状态机并发]** 订单状态机存在并发竞争：Stripe webhook（支付成功）与用户取消、超时关单定时任务可能同时触发，需状态前置校验 + 行级锁/CAS
- **[幂等]** Stripe webhook 必须按 event_id 幂等消费（Stripe 至少一次投递）；下单接口需防重提交（client token 或订单号预生成）
- **[外部集成]** Stripe 沙箱：PaymentIntent 创建、webhook 签名验证、金额货币一致性校验；对象存储：预签名 URL 上传或后端中转上传
- **[定时任务]** 待支付订单超时自动取消并回补库存；闪购活动到期自动下线
- **[异步]** 订单状态变更事件（支付成功/发货）可触发通知；已决策引入 RabbitMQ（decision.md BE-DIM-4，缓存失效/ISR/邮件三类消费者）
- **[权限]** 管理端沿用现有 RBAC @RequirePermission，需为商品/订单/营销/内容新增权限点；消费端订单/地址/购物车/评价必须按 user_id 隔离（防水平越权）
- **[审计日志]** 管理端商品上下架、改价、订单发货、退款审核、内容发布等写操作沿用现有 OperationLog 机制
- **[限流降级]** P95≤100ms / 并发 1000+ 目标下，已决策写接口限流放 Cloudflare WAF 层（decision.md 决策 11）；缓存击穿防护（JetCache penetration protect）
- **[消息队列]** 暂无强需求信号；除非用户要求订单事件解耦，否则单体内事件即可

## 已消化事项（Phase 2.3.1 决策完毕，沉淀于 decision.md）

- [定时任务] 待支付订单超时 30 分钟自动取消并回补库存（BE-DIM-4）
- [限流降级] 写接口限流放 Cloudflare WAF 层，后端不引入限流依赖（决策 11）
- [异步] 订单事件引入 RabbitMQ 解耦（BE-DIM-4，死信/重试/消费幂等）
